## 中英文混排规范化规则

### 规则 1：中英文之间加空格

在中文字符与英文字母/数字之间插入一个半角空格。

```
❌ 研究area的生态学特征
✅ 研究 area 的生态学特征

❌ 使用R语言进行分析
✅ 使用 R 语言进行分析

❌ 共采集了150个样本
✅ 共采集了 150 个样本
```

**例外：** 专有名词中的中英文混合不加空格（如品牌名、固定术语）。

**正则模式：**
```python
import re

def add_cjk_spacing(text):
    # CJK字符后跟英文/数字
    text = re.sub(r'([一-鿿])([A-Za-z0-9])', r'\1 \2', text)
    # 英文/数字后跟CJK字符
    text = re.sub(r'([A-Za-z0-9])([一-鿿])', r'\1 \2', text)
    # 避免重复空格
    text = re.sub(r'  +', ' ', text)
    return text
```

### 规则 2：中文语境使用全角标点

在中文句子中，标点符号应使用全角。

```
❌ 包括土壤,水分和温度
✅ 包括土壤、水分和温度

❌ 结果表明(见表1),该方法有效
✅ 结果表明（见表1），该方法有效

❌ 例如:温度和降水
✅ 例如：温度和降水
```

**标点对照表：**

| 半角 (英文) | 全角 (中文) | 使用场景 |
|------------|-----------|---------|
| , | ，| 中文句中逗号 |
| . | 。| 中文句末句号 |
| : | ：| 中文冒号 |
| ; | ；| 中文分号 |
| ! | ！| 中文感叹号 |
| ? | ？| 中文问号 |
| ( ) | （）| 中文括号 |
| " " | ""  | 中文双引号 |
| ' ' | ''  | 中文单引号 |
| — | —— | 中文破折号（两个） |

### 规则 3：英文语境使用半角标点

在纯英文句子、英文括号内容、学名中使用半角标点。

```
❌ DNA，RNA 和蛋白质（英文术语列举应半角）
✅ DNA, RNA 和蛋白质

❌ （Pinus yunnanensis Franch.）
✅ (Pinus yunnanensis Franch.)
```

**判断"英文语境"的规则：**
- 括号内全为英文 → 半角括号 + 半角标点
- 学名（斜体拉丁文）→ 半角括号
- 英文缩写列举 → 半角逗号
- 括号内有中文 → 全角括号

### 规则 4：英文括号前后空格

中文与英文括号之间加空格；全角括号与中文之间不加空格。

```
❌ 方法(method)的选择
✅ 方法 (method) 的选择

❌ 采用随机森林（random forest） 方法
✅ 采用随机森林（random forest）方法
```

### 规则 5：数字与单位之间加空格

数值与英文单位之间加半角空格。

```
❌ 30cm
✅ 30 cm

❌ 25°C（温度特殊：°与数字无空格，C与°无空格）
✅ 25°C（正确，温度是例外）

❌ 100%（百分号特殊：无空格）
✅ 100%（正确，百分号是例外）
```

**例外（不加空格）：**
- 温度：25°C
- 百分比：100%
- 角度：45°
- 货币符号：$100, ¥500

### 规则 6：数字与中文之间不加空格

数字紧跟中文量词或序数词时不加空格。

```
❌ 第 3 组
✅ 第3组

❌ 共 5 个样方
✅ 共5个样方

但：5 个样方（数字在句首或独立时保持空格规则1）
```

### 规则 7：连续英文中的标点

连续英文词组中的标点遵循英文规则（逗号后空格，句号后空格）。

```
❌ e.g.,soil moisture
✅ e.g., soil moisture

❌ i.e. ,the result
✅ i.e., the result
```

## CJK 规范化执行策略

### 执行顺序

按以下顺序执行规范化，避免规则冲突：

```
1. 先处理标点符号（规则2, 3）— 确定全角/半角
2. 再处理空格（规则1, 4, 5, 6）— 在正确标点基础上加空格
3. 最后处理连续英文（规则7）— 修复英文内部标点
```

### 实现方式

使用 `search_and_replace` 的正则模式批量处理。每条规则转化为一个或多个正则替换操作。

**建议分批执行：**
1. 第一批：全角/半角标点修正（约 5-8 个替换操作）
2. 第二批：中英文间距修正（约 3-4 个替换操作）
3. 第三批：数字相关间距（约 2-3 个替换操作）

### 注意事项

- **不修改代码块、URL、邮箱地址** 中的标点和空格
- **不修改参考文献条目** 中的标点（参考文献有自己的格式规范）
- **不修改已有的正确格式** — 只修复不符合规则的地方
- 执行前先用 word-checker 扫描，确认需要修改的位置
- 执行后再用 word-checker 验证修改结果
