# Cross-Reference Check Rules

## 概述

本文件定义 word-checker 在执行交叉引用检查时的完整规则集。

## 检查对象

### 图 (Figures)

**中文引用模式：**
```
图{n}          → 图1, 图2, 图3
图 {n}         → 图 1, 图 2（空格变体）
如图{n}所示    → 如图1所示
见图{n}        → 见图1
(图{n})        → (图1)
（图{n}）      → （图1）
```

**英文引用模式：**
```
Figure {n}     → Figure 1, Figure 2
Fig. {n}       → Fig. 1, Fig. 2
Fig.{n}        → Fig.1（无空格变体）
Figs. {n}-{m}  → Figs. 1-3（范围引用）
(Fig. {n})     → (Fig. 1)
```

**图题模式（定义位置）：**
```
图{n} {title}         → 图1 研究区域概况
图 {n}. {title}       → 图 1. 研究区域概况
Figure {n}. {title}   → Figure 1. Study area
Fig. {n}. {title}     → Fig. 1. Description
```

### 表 (Tables)

**中文引用模式：**
```
表{n}          → 表1, 表2
表 {n}         → 表 1, 表 2
见表{n}        → 见表1
如表{n}所示    → 如表1所示
(表{n})        → (表1)
（表{n}）      → （表1）
```

**英文引用模式：**
```
Table {n}      → Table 1, Table 2
Tables {n}-{m} → Tables 1-3
(Table {n})    → (Table 1)
```

**表题模式（定义位置）：**
```
表{n} {title}         → 表1 不同处理的方差分析结果
表 {n}. {title}       → 表 1. 方差分析结果
Table {n}. {title}    → Table 1. ANOVA results
```

### 公式 (Equations)

**引用模式：**
```
式({n})        → 式(1), 式(2)
式（{n}）      → 式（1）
公式({n})      → 公式(1)
Eq. ({n})      → Eq. (1)
Equation ({n}) → Equation (1)
Eqs. ({n})-({m}) → Eqs. (1)-(3)
```

**公式编号定义模式：**
```
({n})          → (1), (2)（通常在行末右对齐）
```

### 章节引用

**中文引用模式：**
```
第{n}章        → 第1章, 第2章
第{n}节        → 第1节
{n}.{m}节      → 2.1节
见{n}.{m}      → 见2.1
```

**英文引用模式：**
```
Section {n}    → Section 1, Section 2.1
Chapter {n}    → Chapter 1
```

## 检查规则

### R1: 引用目标存在性

**规则：** 正文中引用的每个图/表/公式编号，在文档中必须存在对应的图题/表题/公式编号定义。

```
扫描策略：
1. 提取所有"定义"（图题/表题/公式编号）→ 得到已定义编号集合
2. 提取所有"引用"（正文中的引用）→ 得到引用编号集合
3. 引用 - 定义 = 悬空引用（dangling references）→ 报告为错误
```

### R2: 编号连续性

**规则：** 图/表编号必须从 1 开始，连续递增，无跳跃。

```
检查逻辑：
已定义编号排序后，应为 [1, 2, 3, ..., n]
如果实际是 [1, 2, 4]，则报告"缺少编号 3"
如果实际是 [1, 2, 2, 3]，则报告"编号 2 重复"
```

### R3: 编号唯一性

**规则：** 同类资产中不得有重复编号（如两个"表2"）。

### R4: 首次引用顺序

**规则：** 在正文中，图/表应按编号顺序首次出现。即图1的首次引用应在图2的首次引用之前。

```
检查逻辑：
对每个编号记录"首次引用位置"（段落号）
按编号排序后，首次引用位置应单调递增
如果图3首次出现在P50，图1首次出现在P65，则报告"引用顺序错误"
```

### R5: 引用格式一致性

**规则：** 全文中同类引用应使用一致的格式。不应混用"图1"和"Figure 1"。

```
检查逻辑：
统计每种格式的出现次数
如果 "图" 出现 15 次，"Figure" 出现 2 次 → 报告不一致
主要格式为"图"，少数"Figure"可能是遗漏
```

### R6: 孤立资产

**规则：** 每个图/表定义在正文中应至少被引用一次。

```
检查逻辑：
定义 - 引用 = 孤立资产（orphan assets）→ 报告为警告（非错误）
```

### R7: 范围引用有效性

**规则：** 范围引用（如"图1-3"或"Tables 1-3"）中的范围应有效。

```
检查逻辑：
起始编号 < 结束编号
范围内所有编号都存在定义
```

## 检查执行流程

```
1. 用 get_document_text 获取全文（此场景确实需要全文）
2. 用正则提取所有"定义"位置和编号
3. 用正则提取所有"引用"位置和编号
4. 按 R1-R7 逐条检查
5. 按严重程度分类报告：
   - ❌ ERROR: R1(引用不存在), R2(编号不连续), R3(编号重复)
   - ⚠ WARNING: R4(引用顺序), R5(格式不一致), R6(孤立资产)
   - ℹ INFO: R7(范围引用建议)
```

## 正则表达式参考

```python
# 中文图引用
re.findall(r'(?:如)?图\s*(\d+)', text)

# 英文图引用
re.findall(r'(?:Figure|Fig\.)\s*(\d+)', text)

# 中文表引用
re.findall(r'(?:如|见)?表\s*(\d+)', text)

# 英文表引用
re.findall(r'Table\s*(\d+)', text)

# 公式引用
re.findall(r'(?:式|公式|Eq\.|Equation)\s*[（(]\s*(\d+)\s*[)）]', text)

# 图题定义（通常在段落开头）
re.findall(r'^图\s*(\d+)[\s.．]', text, re.MULTILINE)
re.findall(r'^(?:Figure|Fig\.)\s*(\d+)[\s.]', text, re.MULTILINE)

# 表题定义
re.findall(r'^表\s*(\d+)[\s.．]', text, re.MULTILINE)
re.findall(r'^Table\s*(\d+)[\s.]', text, re.MULTILINE)
```
